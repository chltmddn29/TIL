# http(HyperText Transfer Protocol)

## http란

- 웹 상에서 데이터를 주고받기 위한 프로토콜
- 클라이언트와 서버 간의 request와 response를 통해 작동
- 다양한 종류의 데이터를 전송할 수 있게 만든 프로토콜
- TCP/IP 프로토콜 사용

## http의 단점

- 데이터를 사람이 볼수 있는 정보로 전송해 데이터 변조의 위험
- 중간에서 패킷을 가로챌수 있고 수정도 가능하기에 보안에 취약
- 이래서 나온것이 **https**
- **https** 에서는 세션 데이터가 암호화된 상태로 전송

## http 흐름

1. 사용자가 웹 브라우저의 도메인을 입력
2. 브라우저는 입력된 도메인 이름을 IP 주소로 변환하기 위해 DNS 서버에 요청 
3. 브라우저는 받은 IP를 통해 서버와 TCP 연결
4. 연결 시 HTTP 요청 메시지를 서버에 보냄
5. 서버가 요청을 처리, HTTP 응답 전송
6. 응답을 받아 구글 웹 사이트에 표시

## HTTP 요청 메서드

1. GET : 정보를 요청(조회)
2. HEAD : 헤더 정보만 요청
3. POST : 서버에 데이터를 전송
4. PUT : 정보를 업데이트(수정)
5. DELETE : 정보 삭제
6. OPTIONS : 서버에 지원하는 HTTP 메서드의 목록 요청
7. PATCH : 리소스의 일부분 수정
