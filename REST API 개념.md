## RESTful API

## REST API란?
- 웹 기반 통신에서 자원을 정의
- 자원에 대한 주소를 사용하여 정보를 주고받는 통신 방법
- 클라이언트와 서버간의 통신을 위한 아키텍처 중 하나
- 주로 HTTP 프로토콜 이용, 데이터 송수신, 자원의 상태를 CRUD 작업을 통해 관리

## REST API의 특징

- stateless 통신 : 서버가 클라이언트의 상태 저장 X, 각각의 요청이 독립적이고 이전 요청의 정보를 기반하지 않음
- 캐시 가능 : HTTP의 기본적인 캐싱 사용 가능, 네트워크 대역폭 사용을 줄이고 성능을 올림
- 계층화 : 클라이언트가 중간 서버를 통해 서버에 접근
- 통합 인터페이스 : 애플리케이션과 서버 간의 통신을 위한 표준 방식 제공, 다양한 클라이언트에서 서버 자원에 접근 가능

## 중요 개념

### 리소스

1. URI로 정의
2. 리소스는 처리되는 대상의 의미
3. URI은 특정 자원의 위치를 나타내는 주소로 HTTP나 FTP 프로토콜과 함께 사용

### 메서드

1. HTTP 메서드로 정의
2. 이 메서드들을 각각 CRUD 연산과 매핑

### 표현

1. HTTP 메서드와 URI로는 원하는 내용을 모두 표현 불가

## URL과 URI의 차이

- URL : 클라이언트가 서버에 특정 자원 요청을 보낼때 사용, 위치 지정
- URI : 리소스를 식별하는 용어, URL의 하위 개념, 리소스의 위치를 지정하는 특별한 형태
