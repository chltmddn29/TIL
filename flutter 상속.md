# flutter 상속

## 1. extends

- 다중 상속 불가
- 부모 클래스의 메서드를 모두 구현하지 않아도 된다.

## 2. **implements**

- 다중 상속 가능
- 부모 클래스의 메서드를 모두 구현해야 한다.

## 3. **with**

- extends, implements의 특징을 합침
- 다중 상속 가능
- 부모 클래스의 메서드를 모두 구현하지 않아도 된다.
- with은 mixin 키워드만 상속 가능

## 4. mixin

- 상속없이 클래스 안에서 코드를 재사용 가능
- 선언시 with 키워드를 사용해야만 가능
